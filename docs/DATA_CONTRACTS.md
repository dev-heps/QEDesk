# QEDesk Data Contracts

QEDesk v0.2 treats proof auditing as a data workflow. This document fixes the
minimum file contracts so the project does not depend on one model's memory or
style.

## Source Files

| Path | Required | Role |
| --- | --- | --- |
| `src/main.tex` | yes | Student-facing natural-language proof note. This is the pedagogical source of truth. |
| `src/Proof.lean` | yes | Lean formalization and verification scratchpad. |
| `blueprint/src/content.tex` | generated | Lean Blueprint-compatible content generated from checked Lean declarations. |
| `build/qedesk-dag.json` | generated | Machine-readable QEDesk DAG sidecar generated by `./bin/qedesk sync`. |
| `build/qedesk-dag.mmd` | generated | Mermaid rendering of the same QEDesk DAG for quick inspection. |
| `schemas/qedesk-dag.schema.json` | yes | JSON schema for QEDesk DAG sidecars. |

## Sync Rule

QEDesk keeps `src/main.tex` as the editable student note. QEDesk node markers
in that file are the editable source of the proof graph. Lean Blueprint source
is generated at `blueprint/src/content.tex`. Lean Blueprint web output is
generated under `blueprint/web/`.

The implemented rule is one-way:

```text
src/main.tex QEDesk markers -> blueprint/src/content.tex
src/main.tex QEDesk markers -> build/qedesk-dag.json
src/main.tex QEDesk markers -> build/qedesk-dag.mmd
src/Proof.lean -> generated Lean map in src/main.tex
src/Proof.lean -> Lean declaration status in blueprint/DAG nodes
```

Manual edits in generated Blueprint files may be overwritten by
`./bin/qedesk sync`. QEDesk does not promise automatic bidirectional sync.

## Blueprint Markers

When possible, LaTeX proof blocks should use explicit markers before relying on
NLP inference:

```tex
% QEDesk[node=mem_power_set][kind=lemma][title=Membership in the power set][lean=mem_power_set][status=checked]
% QEDesk[uses=def_power_set]
```

`qedesk sync` translates those comments into Lean Blueprint-compatible macros
such as `\lean`, `\leanok`, and `\uses`. QEDesk comments are deliberately valid
LaTeX comments and will not break normal PDF builds.

## Gap Taxonomy

Use these labels for natural-language proof gaps:

| Label | Meaning |
| --- | --- |
| `quantifier-jump` | A universal/existential claim changes scope or direction without justification. |
| `missing-hypothesis` | A required assumption is absent. |
| `missing-justification` | A conclusion is asserted without naming the definition, lemma, theorem, or calculation that supports it. |
| `false-inference` | The inference is mathematically false. |
| `undefined-symbol` | A symbol, variable, function, or set is introduced without definition. |
| `ambiguous-expression` | The expression can be read in multiple incompatible ways. |
| `set-membership-gap` | Membership or subset conditions are used incorrectly or implicitly. |
| `calculation-gap` | Algebraic, numerical, or symbolic calculation skips necessary steps. |
| `visual-dependence` | The proof relies on a diagram or intuition without a formal bridge. |
| `statement-drift` | The formalized target no longer matches the natural-language claim. |
| `unspecified` | The agent cannot classify the issue confidently. |

## Hint Levels

QEDesk should expose the smallest useful hint.

| Level | Output |
| --- | --- |
| 0 | Highlight the suspicious span only. |
| 1 | Name the gap type and missing condition. |
| 2 | State a local subgoal in natural language. |
| 3 | Suggest a theorem, lemma, or tactic family. |
| 4 | Suggest a Lean candidate patch. The candidate must still be checked by Lean. |

## DAG Sidecar

The DAG sidecar is a QEDesk-defined artifact, not an upstream Lean Blueprint
standard. It should usually be generated at `build/qedesk-dag.json` or attached
to a review bundle.

Required top-level keys:

- `document`
- `nodes`
- `edges`
- `gaps`

See `schemas/qedesk-dag.schema.json`.

## Verification Report

A verifier report should include:

- Lean build status;
- checked declarations;
- accepted or rejected candidate patches;
- remaining `sorry` blocks;
- axioms or trusted assumptions if available;
- unused or load-bearing hypotheses when known;
- residual risks in natural-language prose.

Lean remains the final verifier. Reports summarize evidence; they do not
replace kernel checking.
